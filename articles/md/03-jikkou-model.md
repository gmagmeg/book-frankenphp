# 高速＆高パフォーマンスを支える技術

この章では、FrankenPHPの高速＆高パフォーマンスを支えるWorkerモードとOctaneについて説明します。

## Workerモードとは
FrankenPHPには２つのモードが用意されており、１つはシングルプロセス、シングルスレッドのPHPとして振る舞うモードです。もう一つがこの章でも取り扱う、１プロセス内でマルチスレッド動作するWorkerモードです。Workerモードはマルチスレッドで動作させることで、高いパフォーマンスを実現しています。

### プロセス・スレッドの違い

Workerモードを理解するために、まずプロセスとスレッドの違いを整理します。

 - **プロセス**
OSが独立したメモリ空間を割り当てる実行単位で、プロセス間でメモリは共有されません。メモリが独立しているため、処理間で変数の内容が干渉し合わないという安全性の高さがメリットです。一方でプロセスの起動コストは高いです。

 - **スレッド**
1つのプロセス内で並行して動く実行単位で、同じプロセスのメモリ空間を共有します。スレッドはプロセスに比べて起動コストが低く、データの受け渡しが容易という特性があります。反面、適切な排他制御を行わないと、別のリクエストの処理によって変数が意図せず上書きされる危険性があります。

**表：プロセスとスレッドの比較**

| 比較項目 | プロセス | スレッド |
|---|---|---|
| メモリ空間 | 独立（共有なし） | 同一プロセス内で共有 |
| 起動コスト | 高い | 低い |
| データ受け渡し | 専用の仕組みが必要 | 容易 |
| 変数の独立性 | 高い（干渉なし） | 低い（排他制御が必要） |

![シングルプロセス・マルチスレッド](images/single-process-thred.png)

通常のPHPはシングルプロセス・シングルスレッドで動作しますが、マルチスレッド環境に向けて排他制御が施されたスレッドセーフ（TS）版のPHPも存在します。FrankenPHPは内部でこのTS版PHPを使用することで、安全なマルチスレッド実行環境を実現しています。

**表：NTS版とTS版（スレッドセーフ版）の比較**

| 比較項目 | NTS版（通常のPHP） | TS版（スレッドセーフ） |
|---|---|---|
| スレッドセーフ | 非対応 | 対応。排他制御あり |
| マルチスレッド動作 | 不可 | 可能 |
| 主な用途 | CGI / PHP-FPM | FrankenPHPのWorkerモード、Swoole等 |
| パフォーマンスオーバーヘッド | 低い | 排他制御のコストの分、少し高い |

## Workerモードの仕組み

Workerモードは、前述のTS版PHPを利用してマルチスレッドで動作するFrankenPHPの実行モードです。
このWorkerモードを有効にするには複雑な設定は不要で、起動時に `--workers` オプションでワーカー数を指定するだけです。

```bash
php artisan octane:frankenphp \
  --workers=4
```

しかし、LaravelはスレッドセーフなTS版PHPを前提として設計されていないため、このままWorkerモードで動作させると、予期せぬ不具合やパフォーマンスの悪化を招く可能性があります。
この問題を解決するのがLaravelの拡張パッケージ、Octaneです。

## Octaneの役割
**Octaneの導入**
前述した通り、LaravelはそのままではWorkerモードに対応していません。この問題を解決するのがOctaneです。Octaneの導入は他のLaravelパッケージと同様に、composerコマンドとartisanコマンドで行えます。

```bash
composer require laravel/octane
php artisan octane:install --server=frankenphp
```

続いて、次のコマンドでサーバーを起動します。

```bash
php artisan octane:frankenphp
```

これでOctaneの導入は完了です。

**Octane導入後の仕組み**

OctaneはLaravelアプリケーションをWorkerモードで安全に動作させるためのWorkerスクリプトを提供します。FrankenPHPの起動時にこのWorkerスクリプトが読み込まれ、アプリケーションの初期化処理（フレームワークの起動・設定読み込みなど）を一度だけ行った後、リクエストを受け付けるループに入ります。

通常のPHP（CGI/FPMモード）では、リクエストごとにPHPプロセスが起動し、フレームワークや設定ファイルを毎回読み込む必要があります。これに対してOctane＋Workerモードでは、初期化コストがアプリケーション起動時の1回のみに抑えられるため、リクエストあたりの処理時間を大幅に短縮できます。

また、複数のWorkerスレッドが並行して実行されるため、同時に複数のリクエストを処理することも可能です。スレッドが同一プロセスのメモリ空間を共有するという特性を活かし、キャッシュや設定データをWorker間で共有することで、メモリ使用量の削減にも貢献します。

FrankenPHPでLaravelを利用する際は、ぜひOctaneを導入しましょう。

## この章のまとめ

この章では、FrankenPHPの高パフォーマンスを支えるWorkerモードと、LaravelでWorkerモードを活用するためのOctaneについて説明しました。

- **Workerモード**：FrankenPHPのTS版PHPを用いたシングルプロセス・マルチスレッドの実行モードです。プロセスより起動コストの低いスレッドを活用し、複数リクエストを並行処理します。
- **Octane**：LaravelをWorkerモードで安全に動作させるための拡張パッケージです。Workerスクリプトを通じてアプリケーションの初期化を一度だけ行い、リクエストごとの初期化コストを大幅に削減します。

FrankenPHPとOctaneを組み合わせることで、通常のPHP実行環境と比べて高いスループットと低いレイテンシを実現できます。
